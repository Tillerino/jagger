# Lombok

Jagger and Lombok work together exactly as you would expect: Accessors and constructors generated by Lombok are
picked up by Jagger. In fact, Lombok is used in Jagger's tests to cut down on boilerplate code:

```java
// ../jagger-tests/base/src/main/java/org/tillerino/jagger/tests/model/features/RequiredPropertyModel.java#L16-L25

@Data
@AllArgsConstructor
@NoArgsConstructor
class RequiredPropertyWithSetter {
    @JsonProperty(required = true)
    private String name;

    @JsonProperty(required = false)
    private String description;
}
```

However, it is important that you order Lombok before Jagger in the annotation processors:

```xml
<!-- ../jagger-tests/pom.xml#L83-L103 -->

<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <annotationProcessorPaths>
            <annotationProcessorPath>
                <groupId>org.projectlombok</groupId>
                <artifactId>lombok</artifactId>
                <version>${lombok.version}</version>
            </annotationProcessorPath>
            <annotationProcessorPath>
                <groupId>org.tillerino.jagger</groupId>
                <artifactId>jagger-processor</artifactId>
                <version>${project.version}</version>
            </annotationProcessorPath>
        </annotationProcessorPaths>
        <compilerArgs>
            <arg>-parameters</arg>
        </compilerArgs>
    </configuration>
</plugin>
```
